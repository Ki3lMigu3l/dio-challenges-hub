## Challenge Overview

This directory references the solution developed for the
Digital Innovation One (DIO) DevOps challenge:

**Docker: Practical Use in a Microservices Scenario**

The challenge focuses on demonstrating containerization
concepts, service isolation, scalability, and reverse proxy
configuration using Docker.



## Solution Repository

The full implementation is maintained in a dedicated repository:

**Project Link**  
https://github.com/Ki3lMigu3l/toshiro-shibakita


## Implemented Concepts

- Containerized microservices architecture  
- Reverse proxy with NGINX  
- PHP-FPM application container  
- MySQL database integration  
- Horizontal scaling  
- Load balancing validation
